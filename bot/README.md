# Telegram Admin Bot - Admin Panel

Полнофункциональная админ панель для управления системой заявок через Telegram бот.

## Функциональность

### 1. 📦 Модуль "Заявки"
- **Просмотр**: Фильтры по статусам (Новые, В работе, Завершенные)
- **Карточка заявки**: ФИО, телефон, email, организация, сфера/вид, статус, дата
- **Действия**: 
  - ✏️ Редактирование полей (ФИО, телефон, email, организация)
  - 🗑 Удаление с подтверждением
  - 📝 Смена статуса
  - 💼 Вынесение решения (привязка тарифа)
- **Создание**: Пошаговое заполнение формы

### 2. 🏷 Модуль "Сферы деятельности"
- Просмотр списка с пагинацией
- Добавление новой сферы
- Редактирование названия
- Удаление (с проверкой) 

### 3. 🔧 Модуль "Виды деятельности"
- Аналогично сферам
- Привязка к сфере при создании

### 4. 💰 Модуль "Тарифы"
- Просмотр (название, описание, цена)
- Добавление (название → описание → стоимость)
- Редактирование всех полей
- Удаление

### 5. ✅ Модуль "Решения"
- Создание решения для заявки
- Выбор тарифа из списка
- Ввод описания
- Просмотр решений по ID заявки

### 6. 📊 Поиск и фильтрация
- **По ID**: Быстрый поиск заявки
- **По телефону**: Поиск клиентов
- **По email**: Поиск по почте
- **По организации**: Поиск по названию
- **По дате**: Фильтр (день/неделя/месяц)

### 7. 📌 Уведомления
- Автоматическое уведомление о новой заявке
- Кнопки: 👀 Посмотреть / ✏️ Редактировать

## Архитектура

```
bot/
├── admin_bot.py          # Основной бот с FSM и обработчиками
├── api_client.py         # Async HTTP клиент для API
├── config.py             # Конфигурация и константы
├── keyboards.py          # Все клавиатуры (inline и reply)
├── utils.py              # Утилиты форматирования и поиска
└── requirements.txt      # Зависимости
```

## Технические особенности

✅ **FSM (Finite State Machine)** - для многошаговых форм
✅ **Async/await** - для асинхронной работы с API
✅ **Пагинация** - для длинных списков (по 5 элементов)
✅ **Inline-кнопки** - для быстрых действий
✅ **Обработка ошибок** - понятные сообщения об ошибках
✅ **Валидация** - проверка email, телефона и формата данных
✅ **Контекст пользователя** - сохранение состояния во время работы

## Установка и запуск

### 1. Установка зависимостей
```bash
cd bot
pip install -r requirements.txt
```

### 2. Конфигурация
Отредактируйте `config.py`:
```python
TOKEN = "ВАШ_TELEGRAM_BOT_TOKEN"         # Получить у @BotFather
ADMIN_CHAT_ID = ВАШ_ID                    # Ваш Telegram ID
API_BASE_URL = "http://localhost:5000"    # URL вашего API
```

### 3. Запуск бота
```bash
python admin_bot.py
```

## Интеграция с API

Бот использует REST API FormAPI:

### Endpoints
- `GET/POST/PUT/DELETE /api/applications` - Заявки
- `GET/POST/PUT/DELETE /api/sphereActivity` - Сферы
- `GET/POST/PUT/DELETE /api/typeActivity` - Виды
- `GET/POST/PUT/DELETE /api/tarif` - Тарифы
- `GET/POST/PUT/DELETE /api/solution` - Решения

### Асинхронный клиент
```python
from api_client import client

# Получить все заявки
apps = await client.get_applications()

# Создать заявку
result = await client.create_application({...})

# Обновить заявку
await client.update_application(app_id, {...})

# Удалить заявку
await client.delete_application(app_id)
```

## Потоки работы

### Просмотр заявок
1. Нажать 📦 Заявки
2. Выбрать фильтр статуса
3. Просмотреть карточку
4. Выбрать действие (редактирование, удаление, статус, решение)

### Создание решения
1. Нажать ✅ Решения → Новое решение
2. Выбрать или найти заявку
3. Выбрать тариф
4. Ввести описание
5. Сохранить

### Поиск
1. Нажать 📊 Поиск
2. Выбрать критерий (ID, телефон, email, организация, дата)
3. Ввести значение
4. Получить результаты

## Структура FSM (Finite State Machine)

```
MAIN_MENU
├─ APPS_MENU
│  ├─ APPS_VIEW (просмотр)
│  ├─ APPS_CREATE (создание)
│  ├─ APPS_EDIT (редактирование)
│  └─ APPS_STATUS_CHANGE
├─ SPHERES_MENU → SPHERES_VIEW/CREATE/EDIT
├─ TYPES_MENU → TYPES_VIEW/CREATE/EDIT
├─ TARIFFS_MENU → TARIFFS_VIEW/CREATE/EDIT
├─ SOLUTIONS_MENU → SOLUTIONS_CREATE/VIEW
└─ SEARCH_MENU → SEARCH_BY_*
```

## Обработка ошибок

- ❌ API недоступен → "Timeout: API не ответил"
- ❌ Неверный формат → Валидация перед отправкой
- ❌ Запись не найдена → "Заявка не найдена"
- ✅ Успех → "✅ Действие выполнено"

## Расширение функциональности

### Добавить новый фильтр
1. Добавить состояние в `config.py` (States)
2. Создать обработчик в `admin_bot.py`
3. Добавить кнопку в `keyboards.py`
4. Реализовать логику в `utils.py`

### Добавить новый модуль
1. Создать API методы в `api_client.py`
2. Добавить FSM состояния
3. Создать обработчики
4. Создать клавиатуры

## Примечания

- 🔒 Убедитесь, что боту разрешен доступ к вашему API
- 🗄️ API должен быть запущен перед стартом бота
- ⏱️ Timeout установлен на 10 секунд
- 📝 Пагинация по 5 элементов (редактируется в `config.py`)
- 🔑 Используется StateMemoryStorage (для продакшена рекомендуется Redis)

## Troubleshooting

### Бот не отвечает
```bash
# Проверить токен в config.py
# Убедиться, что интернет есть
# Проверить права доступа токена у @BotFather
```

```bash
# Проверить URL в config.py
# Убедиться что API запущен: curl http://localhost:5000/api/applications

```

### FSM не работает
```bash
# StateMemoryStorage хранит состояние в памяти
# При перезагрузке бота состояния теряются
# Для продакшена используйте Redis Storage
```
