## 🏗️ АРХИТЕКТУРА

### FSM (Finite State Machine)

```
┌─────────────────┐
│   MAIN_MENU     │
└────────┬────────┘
         │
    ┌────┴────────────────────┐
    │                         │
┌───▼──┐  ┌───────┐  ┌──────┐
│APPS  │  │SPHERE │  │TYPES │
└───┬──┘  └──┬────┘  └──┬───┘
    │         │        │
┌───▼───────────────────▼──┐
│ EDIT, CREATE, DELETE,    │
│ SEARCH, FILTER, PAGINATE │
└──────────────────────────┘
```

### API Integration

```
Bot (telebot) ──async──► API Client (aiohttp)
                            │
                            ▼
                      FormAPI (.NET)
                            │
                            ▼
                    SQL Server (FormDB)
```

### Порядок выполнения команды

```
1. Пользователь нажимает кнопку
   ↓
2. Callback обработчик получает событие
   ↓
3. FSM устанавливает новое состояние
   ↓
4. Async функция выполняет запрос к API
   ↓
5. Ответ форматируется и отправляется пользователю
   ↓
6. Bot ждет следующего действия пользователя
```

## 🔄 Обновление и расширение

### Добавить новый модуль

1. Добавить API методы в `api_client.py`
2. Добавить FSM состояния в `config.py`
3. Добавить обработчики в `bot_extended.py`
4. Добавить UI в `keyboards.py`
5. Обновить `DOCUMENTATION.md`

### Интегрировать с вебхуками

```python
# В bot_extended.py
from fastapi import FastAPI, Request
from notifications import handle_webhook_new_app

app = FastAPI()

@app.post("/webhook/new-application")
async def webhook(request: Request):
    data = await request.json()
    await handle_webhook_new_app(data)
    return {"status": "ok"}
```

## ⚠️ Известные ограничения

1. **StateMemoryStorage** - состояние теряется при перезагрузке
   - Решение: Использовать RedisStorage

2. **Пулинг** - менее эффективен чем вебхуки
   - Решение: Использовать webhook для продакшена

3. **Пагинация** - фиксированная на 5 элементов
   - Решение: Изменить ITEMS_PER_PAGE в config.py

4. **Асинхронность в обработчиках**
   - Решение: Использовать asyncio.run() (текущая реализация)
